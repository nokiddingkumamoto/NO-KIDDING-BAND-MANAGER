NO KIDDING BAND MANAGER

V9.0.4 PUBLIC
GitHub・Cloudflare共有設定手順

このファイルをChrome・Safari・Edgeなどのブラウザで開いてください。設定作業はスマートフォンから行えます。

1．GitHubへアップロード

  1. GitHubの NO-KIDDING-BAND-MANAGER リポジトリを開きます。
  2. V9.0.4 PUBLICのZIPを解凍します。
  3. 解凍したフォルダの中身を、リポジトリの最上段へアップロードします。
  4. functions フォルダと、その中の api フォルダも階層を変えずにアップロードします。
  5. ZIPファイル自体はGitHubへアップロードしません。

2．Cloudflare PagesをGitHubへ接続

  1. Cloudflareダッシュボードを開きます。
  2. Workers & Pages を開きます。
  3. Create application → Pages → Connect to Git を選びます。
  4. GitHubの NO-KIDDING-BAND-MANAGER リポジトリを選びます。
  5. Framework presetは None、Build commandは空欄、Build output directoryは / または . にします。
  6. Save and Deploy を押します。

すでに同じGitHubリポジトリと接続済みのPagesがある場合は、新規作成せず、そのプロジェクトを使います。

3．D1データベースを作成

  1. Cloudflareで Storage & Databases → D1 SQL database を開きます。
  2. Create database を押します。
  3. Database nameを no-kidding-band-manager にして作成します。
  4. 作成したデータベースの Console を開きます。
  5. 同梱の schema.sql を開き、中身をすべてコピーしてConsoleへ貼り付けます。
  6. Execute を押します。

4．PagesとD1を接続

  1. Workers & Pages からNO KIDDINGのPagesプロジェクトを開きます。
  2. Settings → Bindings を開きます。
  3. Add binding → D1 database を選びます。
  4. Variable nameを必ず DB にします。
  5. D1 databaseで no-kidding-band-manager を選び、保存します。

5．再デプロイ

  1. Cloudflare Pagesの Deployments を開きます。
  2. 最新のデプロイで Retry deployment を実行します。
  3. 発行された pages.dev のURLを開きます。
  4. ログイン画面なしでトップページが表示されれば完了です。

6．Androidのホーム画面へ追加

  1. ChromeでアプリのURLを開きます。
  2. 右上の︙を押します。
  3. 「ホーム画面に追加」または「アプリをインストール」を押します。

7．iPhoneのホーム画面へ追加

  1. SafariでアプリのURLを開きます。
  2. 共有ボタンを押します。
  3. 「ホーム画面に追加」を押します。

8．共有動作を確認

  1. 1台目で予定を1件登録します。
  2. 2台目で同じURLを開きます。
  3. 10秒以内に同じ予定が表示されることを確認します。
  4. 2台目で在庫を変更し、1台目にも反映されることを確認します。
